data-gallery-root lang="ru" data-v-aad06d04><div data-v-aad06d04><!--[--><!--]--></div><div id="post-content-body" data-v-aad06d04><div><div class="article-formatted-body article-formatted-body article-formatted-body_version-1"><div xmlns="http://www.w3.org/1999/xhtml">В статье «<a href="https://habrahabr.ru/company/efo/blog/312424/">Базовые станции Multi-Tech позволяют развернуть сеть LoRaWAN за пару кликов</a>» говорилось о том, как просто создать сеть IoT, используя оборудование Multi-Tech. Теперь настало время запустить наше первое приложение.<br/>
<br/>
<div style="text-align:center;"></div><br/>
<a name="habracut"></a><br/>
Итак, нам понадобится:<br/>
<br/>
<ul>
<li><a href="http://wless.ru/technology/?action=details&amp;id=676&amp;pf=tech&amp;pf_id=9&amp;prod=12&amp;tech=9&amp;type=156"><b>шлюз Conduit</b> (AEP)</a> с установленным модулем <b>mCard (MTAC-LORA-868)</b></li>
<li>антенна диапазона 868 МГц</li>
<li>компьютер (ПК)</li>
</ul> <br/>
ПК мы подключим через Ethernet к шлюзу и будем использовать для настройки/конфигурации последнего.<br/>
<br/>
<b>Шлюз Conduit</b> – это по сути та же базовая станция, но для использования внутри помещений (исполнение indoor):<br/>
<br/>
<br/>
<br/>
Для начала нам нужно будет настроить <b>LoRaWAN Network Server</b>, а потом создать наше приложение в графической среде <b>Node-Red</b>. Приложение будет принимать сообщения от оконечных устройств LoRaWAN и посылать их обратно (режим эха), попутно отображая принятые сообщения в интерфейсе Node-Red.<br/>
<br/>
Итак, приступим.<br/>
<br/>
По умолчанию в шлюзе установлен фиксированный <b>IP 192.168.2.1, логин – admin, пароль – admin</b>. После успешного ввода пароля сразу переходим в пункт меню <b>Setup -&gt; LoRa Network Server</b> и видим следующую картинку:<br/>
<br/>
<br/>
<br/>
Посмотрим, что тут за что отвечает:<br/>
<br/>
<ol>
<li>Чтобы включить сервер, поставим галочку <b>Enabled</b>.<br/>
 <br/>
 </li>
<li>У шлюза сконфигурированы 3 фиксированных канала шириной 125 кГц (центральные частоты — 868,1, 868,3 и 868,5 МГц), и можно сконфигурировать пять дополнительных каналов шириной 125 кГц и с расстоянием между центральными частотами 200 кГц. Эти дополнительные каналы конфигурируются заданием центральной частоты дополнительного диапазона в поле <b>Additional Channels</b>. Например, если в поле <b>Additional Channels</b> мы задаем значение 869,3, по получаем дополнительные каналы со следующими центральными частотами: 868,9; 869,1; 869,3; 869,5; 869,7 МГц.<br/>
Диапазоны допустимых значений в поле <b>Additional Channels</b> — 863,5-867,5 и 869,1-869,5 (МГц).<br/>
<br/>
</li>
<li>Поле <b>Mode</b> может принимать два значения: <b>NETWORK SERVER</b> и <b>PACKET FORWARDER</b>. Если мы планируем использовать LoRaWAN Network Server на самом шлюзе, то оставляем значение NETWORK SERVER, а если, скажем, в облаке – то PACKET FORWARDER. В нашем случае оставляем <b>NETWORK SERVER</b>.<br/>
 <br/>
 </li>
<li>Галочка <b>Public</b> относится к типу сети – <b>Public</b> или <b>Private</b>. Если используется тип <b>Private</b> (галка не установлена), то быстрее происходит присоединение к сети <b>LoRaWAN</b> оконечных устройств (1-2 сек.), а также при желании можно отключать шифрование полезной нагрузки.<br/>
 <br/>
 </li>
<li>Наиболее важными являются поля <b>Network ID</b> и <b>Network Key</b>. По сути они соответствуют значениям <b>AppEUI</b> и <b>AppKey</b> спецификации LoRaWAN, но так как использование достаточно длинных (8 и 16 байт соответственно) шестнадцатеричных значений не всегда удобно, Multi-Tech решил предоставить пользователям право выбора. В шлюзах/БС Conduit <b>AppEUI</b> и <b>AppKey</b> можно задавать двумя способами: <br/>
 <br/>
а) <b>человеколюбивым</b> – в поле <b>Network ID</b> выбираем <b>Name</b> и ниже пишем человекопонятное имя сети (шлюз сам переведет наше имя в значение <b>AppEUI</b>);<br/>
б) <b>соответствующим спецификации LoRaWAN</b> — в поле <b>Network ID</b> выбираем <b>EUI</b> и записываем шестнадцатеричное значение <b>AppEUI</b>.<br/>
<br/>
Значения <b>AppEUI</b> и <b>AppKey</b> в шлюзе/БС должны соответствовать значениям, запрограммированным в оконечных устройствах LoRaWAN.<br/>
<br/>
</li>
<li>Другие настройки оставим пока без внимания</li>
</ol><br/>
После конфигурирования сервера LoRaWAN нужно не забыть нажать <b>Save and Restart</b>, чтобы применить внесенные изменения.<br/>
<br/>
Вот такая несложная настройка. Всё, теперь можно снова залогиниться и перейти в пункт меню <b>Apps -&gt; Node-Red</b>, чтобы создать демонстрационное приложение.<br/>
<br/>
После повторного ввода логина/пароля admin/admin (это уже для доступа к Node-Red) мы попадаем в <b>Node-Red</b>:<br/>
<br/>
<br/>
<br/>
В левой части находятся узлы, которые мы можем перетаскивать в основное поле и соединять между собой с помощью мышки. Для нашего примера возьмем из раздела <b>Input</b> узел <b>lora</b>, а из раздела <b>Output</b> – узлы <b>debug</b> и <b>lora</b>, и соединим их так, как показано на картинке:<br/>
<br/>
<br/>
<br/>
Нажмем кнопку <b>Deploy</b> в верхнем правом углу окна Node-Red – всё, приложение готово!<br/>
<br/>
<br/>
<br/>
Теперь можно посмотреть, что получается при отправке сообщений с устройства LoRaWAN на шлюз Conduit. После успешного приема сообщения от устройства мы должны получить ответное сообщение на это же устройство, а также можем увидеть это сообщение (Hello_Habroworld!) на вкладке debug в правой части окна Node-Red (благодаря подключенному узлу debug):<br/>
<br/>
<br/>
<br/>
В нашем демонстрационном примере полученные сообщения эхом отправляются обратно, в реальной же конфигурации их можно без дополнительных усилий пересылать на определенные порты TCP/UDP удаленного сервера, посылать в виде сообщений MQTT, SMS, e-mail, сохранять в файл, публиковать в Twitter, обрабатывать на шлюзе/БС и пр. <br/>
<br/>
Конечно, аналогичным образом можно передавать данные и в обратном направлении.<br/>
Таким образом, можно очень быстро создавать приложения IoT и демонстрировать их любимым заказчикам.</div></div></div><!----><!----></div><!----><!----></div><!--]--><!----><div
